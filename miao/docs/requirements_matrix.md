# 比赛规则需求矩阵

## 1. 硬性需求矩阵

| 规则摘要 | 对应技术需求 | 是否硬性 | 对应指标 | 忽略风险 | 推荐实现 |
|---|---|---:|---|---|---|
| 结合唤醒音频的发音人信息 | 从 wake audio 提取目标 speaker embedding | 是 | CER / RR | 退化为普通 ASR，旁人语音会被识别 | CAM++ / ERes2Net / ECAPA 多裁剪融合 |
| 识别远场家居控制指令 | 中文短指令 ASR | 是 | CER | 目标人指令识别错误 | Paraformer / SenseVoice / Zipformer + 指令后处理 |
| 拒识非唤醒说话人 | accept / reject 决策 | 是 | RR | 非目标人误触发 | SV + pVAD + 融合决策 |
| 目标与干扰人同时说话时只识别目标 | 目标说话人检测和提取 | 是 | CER / RR | 输出干扰人文本 | target-speaker pVAD + TSE |
| SNR -5 dB 到 5 dB | 噪声鲁棒性 | 是 | CER | 低 SNR 崩溃 | 噪声增强、混响增强、鲁棒 ASR 微调 |
| 重叠率 0% 到 100% | 多说话人重叠鲁棒性 | 是 | CER / RR | 重叠样本错误严重 | 合成 overlap 数据训练 pVAD / TSE |
| 同一时间最多 2 个说话人 | 二人上限的目标提取 | 是 | CER / 效率 | 使用过重通用分离模型 | 二人混合条件式 TSE |
| 正样本和拒识样本共同评测 | 正负样本分流 | 是 | CER / RR | 单优化 ASR 或单优化拒识都不够 | 按本地综合分数调阈值 |
| JSON 单文件提交 | 格式严格生成 | 是 | 有效提交 | 格式错误导致无效 | schema.py + result writer |
| duration 为 batch=1 总推理时间 | 逐条推理计时 | 是 | 推理效率 | 耗时不可信，代码核查风险 | timer 记录每条样本时延 |
| 提交模型和推理脚本 | 可复现工程 | 是 | 代码核查 | 排名后核查失败 | 固定入口、依赖、配置、随机种子 |

## 2. 底线要求

1. 拒识样本必须输出空字符串。
2. 正样本必须尽量输出目标发音人的文本，而不是干扰人文本。
3. 所有推理必须 batch=1 可运行。
4. 不能写依赖测试集 A 文件名的特殊规则。
5. 提交 JSON 字段必须稳定生成。
6. 代码必须能够在干净环境复现。

## 3. 冲击特等奖的关键差异点

| 差异点 | 价值 |
|---|---|
| 质量感知 speaker embedding | 短 wake、噪声 wake 下更稳 |
| target-speaker pVAD | 解决重叠场景整段 embedding 不可靠问题 |
| 条件式 TSE | 在疑难样本上降 CER，同时控制效率 |
| hard negative 训练 | 提升相似音色和同文本负样本拒识率 |
| 指令级后处理 | 降低短文本 CER |
| 融合决策 | 平衡 CER 和 RR，不靠单一阈值 |
| ONNX / TensorRT / INT8 | 争取效率分 |
