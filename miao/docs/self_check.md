# 自检与修正记录

## 第 1 轮自检：规则覆盖

| 问题 | 风险 | 修正措施 |
|---|---|---|
| 只用 ASR 无法拒识非目标人 | RR 权重 40%，会严重失分 | 加入 speaker verification 和融合拒识决策 |
| 只用整段 speaker similarity 无法处理重叠 | query embedding 被干扰人污染 | 加入 target-speaker pVAD |
| 目标与干扰人 100% 重叠时普通 ASR 输出混合文本 | CER 变高 | 加入条件式 TSE，只提取目标发音人 |
| 命令后处理可能把非目标语音纠成合法命令 | RR 降低 | 后处理必须在身份接收之后启用 |
| 固定阈值对短 wake / 噪声 wake 不稳定 | 泛化差 | 引入 wake quality 和 query SNR 动态阈值 |

修正后：方案从“SV + ASR”升级为“SV + pVAD + 条件式 TSE + ASR + 融合决策”。

## 第 2 轮自检：效率与工程落地

| 问题 | 风险 | 修正措施 |
|---|---|---|
| 全量运行 TSE 推理慢 | 20% 效率分下降 | TSE 只在重叠 / 不确定样本触发 |
| 同时常驻多个大 ASR 模型 | 内存占用高 | 主推理只保留一个 ASR，另一个作 teacher |
| pVAD 如果过大也会拖慢 | batch=1 延迟高 | pVAD 使用小 TDNN / tiny Conformer |
| INT8 量化可能影响 RR | speaker similarity 分布变化 | 量化后重新校准阈值 |
| JSON 格式细节不确定 | 提交失败 | schema.py 做校验，保留有标签 / 无标签兼容 |

修正后：确定“轻量门控 + 条件复杂模型”作为主工程方案。

## 第 3 轮自检：泛化与决赛表达

| 问题 | 风险 | 修正措施 |
|---|---|---|
| 过拟合测试 A | 隐藏 B 掉分 | 构造 speaker-disjoint、noise-disjoint、command-template-disjoint 验证集 |
| 没有消融实验 | 答辩说服力弱 | 设计 ASR only、SV+ASR、+pVAD、+TSE、+后处理、+部署优化消融 |
| 只报告平均分 | 无法证明复杂场景鲁棒 | 按 SNR、overlap、hard negative、短 wake 分桶 |
| 方案看起来像堆模型 | 创新性不突出 | 强调 target-speaker pVAD、质量感知动态阈值、条件式 TSE 和 ASR-aware 训练 |
| 部署说明不足 | 应用价值分不足 | 增加低延迟、低内存、智能家居误触发防护说明 |

修正后：方案定位为“目标说话人身份绑定 + 抗干扰指令识别 + 高效部署”的完整系统。

## 最终自检清单

- [ ] 完整覆盖唤醒音频、待识别音频、拒识标签为空等比赛输入输出；
- [ ] 直接对应 CER、RR、推理效率三大核心指标；
- [ ] 支持 -5 dB 到 5 dB SNR 训练和评估；
- [ ] 支持 0% 到 100% 重叠率增强；
- [ ] 非目标说话人输出空字符串；
- [ ] 复杂模块条件触发，避免推理过慢；
- [ ] 训练 / 验证说话人不重合；
- [ ] 阈值通过本地验证集搜索；
- [ ] JSON 输出通过 schema 校验；
- [ ] 推理入口可在干净环境复现；
- [ ] 决赛材料包含创新性、调优、落地、完整性和消融实验。
