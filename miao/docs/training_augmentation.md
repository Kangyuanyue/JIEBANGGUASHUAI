# 训练与数据增强策略

## 1. 训练数据组织方式

推荐将训练样本组织为 episode：

```text
wake_audio: speaker A 的唤醒音频
query_audio: 目标或非目标待识别音频
label: 正样本为 speaker A 指令文本，负样本为空字符串
meta: snr / overlap_ratio / noise_type / speaker_pair / scenario
```

这样能直接模拟比赛输入输出，而不是只训练普通 ASR。

## 2. 正负样本构造

| 样本类型 | 构造方式 | 目的 |
|---|---|---|
| 正样本 | wake 和 query 来自同一说话人 | 降低目标人 CER |
| 普通负样本 | wake 和 query 来自不同说话人 | 提升 RR |
| 同文本负样本 | 非目标人说相同家居指令 | 防止模型只靠文本合法性接收 |
| 相似音色负样本 | embedding 接近的不同说话人 | 提升 hard negative 拒识能力 |
| 重叠正样本 | target command + interference speech | 训练目标语音识别能力 |
| 重叠负样本 | non-target command + noise / target wake | 训练非目标拒识 |
| 无语音样本 | 静音 / 环境噪声 | 防止空音频误触发 |

## 3. 噪声增强

| 噪声类型 | 示例 | 作用 |
|---|---|---|
| 稳态家居噪声 | 空调、风扇、冰箱 | 贴近比赛非人声噪声 |
| 瞬态家居噪声 | 门声、锅碗声、敲击声 | 提升开放环境鲁棒性 |
| 背景媒体声 | 电视、音乐、播客 | 模拟家庭背景人声 |
| 通用噪声 | 白噪声、粉红噪声、MUSAN | 扩大 SNR 覆盖 |

建议 SNR 采样：

```text
train: [-8, -5, -3, 0, 3, 5, 8] dB 随机采样
validation: 固定 -5 / 0 / 5 dB 分桶
stress: -10 / -8 dB 极端压力测试
```

## 4. 重叠增强

比赛重叠率从 0% 到 100%，因此训练时应固定构造：

```text
overlap_ratio = [0, 0.25, 0.50, 0.75, 1.00]
```

构造时注意：

1. 干扰人既可以说家居指令，也可以说无关语音；
2. 必须包含同文本重叠，即目标和干扰人说相同 / 相近命令；
3. 目标人音量可低于干扰人，模拟远场困难样本；
4. 对 pVAD 训练保留帧级 target / non-target / overlap 标签。

## 5. 远场增强

推荐增强：

- RIR 卷积；
- 随机距离衰减；
- 混响时间 RT60 随机采样；
- 麦克风频响退化；
- 低通 / 高通滤波；
- 随机 gain。

## 6. wake 音频增强

wake audio 是目标身份注册依据，必须单独增强：

| 增强 | 目的 |
|---|---|
| 截短 wake | 适配短唤醒音频 |
| 加噪 wake | 适配带噪唤醒 |
| 混响 wake | 适配远场 wake |
| 音量变化 | 适配说话距离变化 |
| VAD 裁剪误差 | 提升预处理鲁棒性 |

## 7. 训练 / 验证 / 压力测试划分

| 集合 | 划分原则 | 用途 |
|---|---|---|
| Train | 可混合官方开发集、公开数据和增强数据 | 训练 ASR / SV / pVAD / TSE |
| Dev | speaker-disjoint，不能与 Train 共享说话人 | 阈值搜索和早停 |
| Stress | 按 SNR、overlap、noise、hard negative 分桶 | 定位系统短板 |
| Holdout | 不参与任何调参 | 模拟隐藏 B 泛化 |

## 8. 防止过拟合测试 A

1. 不针对测试 A 的单条样本写规则。
2. 阈值在 dev 上搜索，在测试 A 上只验证。
3. 保留独立 holdout。
4. 每次调参输出分桶指标，而不是只看平均分。
5. 使用多随机种子验证阈值稳定性。
6. 不使用任何隐藏测试集信息。
