# 语音转文字（ASR）轻量化及前沿方案研究现状综述


## 一、研究背景与目标

在智能家居、车载系统等边缘场景中，语音指令识别对ASR模型提出三个核心要求：
- **中文精度高**：智能家居指令多为中文，错误识别会导致设备误操作
- **推理速度快**：用户期望近乎实时的响应
- **模型体积小**：边缘设备存储和内存有限，无法承载大模型

本综述聚焦2025-2026年发布的轻量化及前沿ASR方案，从精度、速度、体积、开源生态、**鲁棒性及特定场景优化**等维度进行横向对比，为系统集成提供选型依据。

经调研，筛选出以下十一个代表性方案：

| 方案 | 定位 | 核心特点 |

## 三、各方案详细分析

### 3.1 Qwen3-ASR-0.6B

**方法思路：**
Qwen3-ASR采用大型音频语言模型（Large Audio-Language Model, LALM）范式。与传统ASR的"声学特征→音素→文本"的自下而上的声学模式匹配不同，LALM范式先对音频信号形成高层语义理解，再基于这个理解生成转录文本。

具体来说：
- **音频编码**：基于Qwen3-Omni的音频理解能力，将音频信号编码为深层语义表示
- **语义理解**：利用大语言模型的世界知识和语言建模能力，理解音频的语义内容
- **条件生成**：在高层语义理解的基础上，自回归地生成转录文本

这种范式的优势在于：长语音转录、噪声鲁棒性、专名识别、多语言和方言覆盖等传统ASR的难点问题，都能被更自然地解决。

Qwen3-ASR家族包含三个模型：

| 模型 | 参数量 | 定位 |
| :--- | :--- | :--- |
| Qwen3-ASR-1.7B | 1.7B | 开源ASR模型中的SOTA性能 |
| Qwen3-ASR-0.6B | 0.6B | 最佳精度-效率权衡 |
| Qwen3-ForcedAligner-0.6B | 0.6B | 首个轻量级LALM多语言强制对齐模型 |

**性能指标：**

| 指标 | 数值 |
| :--- | :--- |
| 参数量 | 0.6B |
| 模型大小 | ~1.3GB |
| 中文CER (AISHELL-1) | 3.76% |
| 首Token延迟 (TTFT) | 92ms |
| 吞吐量 (128并发) | 2000秒语音/秒 |
| 支持语言 | 52种（含22种中国方言） |

**配套工具：**
- Qwen3-ForcedAligner-0.6B：词级时间戳对齐，11种语言上优于三大对齐模型
- 原生支持ONNX Runtime部署

**优势**：精度-效率平衡最佳，多语言能力突出，配套工具完善
**局限**：1.3GB在极低端设备（<1GB内存）上仍有压力


### 3.2 SenseVoice（FunAudioLLM）

**基本信息：**
- **发布方**：阿里通义语音团队（Tongyi SpeechTeam）
- **发布时间**：2024年7月
- **论文**：*FunAudioLLM: Voice Understanding and Generation Foundation Models for Natural Interaction Between Humans and LLMs*（arXiv:2407.04051）
- **代码**：https://github.com/FunAudioLLM

**方法思路：**
FunAudioLLM是一个旨在增强人类与大型语言模型之间自然语音交互的模型系列。其核心包含两个创新模型：

| 模型 | 功能 |
| :--- | :--- |
| **SenseVoice** | 多语言语音识别、情感识别、音频事件检测 |
| **CosyVoice** | 自然语音生成，支持多语言、音色、说话风格和说话人身份控制 |

SenseVoice包含两个版本：
- **SenseVoice-Small**：超低延迟ASR，支持5种语言（中文、英语、粤语、日语、韩语），非自回归端到端架构
- **SenseVoice-Large**：高精度ASR，支持50+语言，编码器-解码器架构

**技术架构（SenseVoice-Small）：**
SenseVoice-Small是一个非自回归编码器模型，通过四个特殊任务嵌入（LID、SER、AEC、ITN）统一处理多种语音理解任务。这种设计使其在推理速度上具有显著优势。

### 3.3 Moonshine

**方法思路：**
Moonshine的核心贡献是挑战了"多语言ASR优于单语ASR"的普遍假设。
- **传统观点**：多语言模型可以利用跨语言的语音相似性（跨语言音素共性），因此性能优于单语模型。
- **Moonshine的发现**：对于足够小的模型（27M参数），在高质量人工标注、伪标签和合成数据的精心平衡混合上训练单语系统，性能显著优于多语系统。
- **架构**：编码器-解码器Transformer，采用旋转位置嵌入（RoPE）
- **参数量**：27M（极致轻量）
- **设计目标**：匹配Whisper的精度，同时通过消除零填充来优化计算效率

**性能指标：**

| 指标 | 数值 |
| :--- | :--- |
| 参数量 | 27M |
| 模型大小 | ~100MB |
| 中文CER (AISHELL-1) | ~36% |
| 错误率对比 | 比Whisper Tiny低 48% |
| 跨规模对比 | 优于9倍大的Whisper Small |

**优势**：极致轻量（27M），推理速度极快，Apache 2.0许可证
**局限**：中文精度严重不足（CER 36%），无法用于中文指令识别


### 3.4 FireRedASR2S

**方法思路：**
FireRedASR2S是一个工业级一体化ASR系统，在统一流水线中集成了四个模块：
1. ASR（语音识别）
2. VAD（语音活动检测）
3. LID（语种识别）
4. Punc（标点预测）

所有模块均在评估基准上达到SOTA性能。

**各模块详解：**

**（1）FireRedASR2（ASR模块）**

两个变体：

| 变体 | 参数量 | 定位 | 关键指标 |
| :--- | :--- | :--- | :--- |
| FireRedASR2-LLM | 8B+ | SOTA精度 | 普通话CER 2.89%，方言基准 11.55% |
| FireRedASR2-AED | 1B+ | 精度-效率平衡 | 支持普通话、方言、英语、歌声识别 |

**（2）FireRedVAD（VAD模块）**
- **架构**：基于深度前馈序列记忆网络（DFSMN）
- **参数量**：仅0.6M
- **关键指标**：帧级F1 97.57%，AUC-ROC 99.60%
- **对比**：优于Silero-VAD、WebRTC-VAD

**（3）FireRedLID（语种识别模块）**
- **支持**：100+语言，20+中国方言
- **关键指标**：FLEURS数据集上97.18%准确率

**（4）FireRedPunc（标点预测模块）**
- **关键指标**：平均F1 78.90%，优于FunASR-Punc（62.77%）

**优势**：中文精度公开基准第一（0.57%），VAD极轻量（0.6M），全流程一体化
**局限**：ASR模型较大（1B+），依赖GPU推理


### 3.5 sherpa-onnx

**方法思路：**
sherpa-onnx是基于ONNX Runtime的轻量化语音处理推理框架。
- **技术架构**：推理引擎为ONNX Runtime，模型统一转换为ONNX格式
- **支持的模型**：Whisper、Moonshine、SenseVoice、Qwen3-ASR、Paraformer等
- **跨平台**：x86、ARM、RISC-V，覆盖Linux/Windows/macOS/Android/iOS
- **语言API**：C++、Python、Rust、Java等12种

**优势**：统一部署框架，一次集成多模型复用，NPU加速支持
**局限**：本身不是ASR模型，需配合具体模型使用


### 3.6 FormalASR（端到端中文口语转书面语）

**方法思路：**
FormalASR提出端到端解决方案，直接通过单个紧凑模型（0.6B和1.7B）将中文口语转录为正式的书面文本，**推理时无需任何辅助LLM**。
- **数据集构建**：利用DeepSeek-V3.2重写逐字转录，构建WenetSpeech-Formal和Speechio-Formal数据集
- **训练策略**：基于Qwen3-ASR进行全参数监督微调（SFT）

**性能指标：**

| 指标 | FormalASR-0.6B | FormalASR-1.7B |
| :--- | :--- | :--- |
| 域内CER降低 (WenetSpeech) | 31.4% | 34.7% |
| 跨域CER降低 (Speechio) | 28.8% | 37.4% |
| BF16模型大小 | 1.46 GB | 3.80 GB |
| Q8_0量化大小 | 0.78 GB | 2.03 GB |
| Q4_K量化大小 | 0.42 GB | 1.08 GB |

**技术特点：**
- 去除填充词，输出Token减少约22.8%
- 支持GGUF格式量化，端侧友好
- 单模型部署，无需云端LLM API


### 3.7 Fun-ASR（基于LLM与强化学习的工业级ASR）

**方法思路：**
Fun-ASR是阿里通义团队推出的大规模基于LLM的ASR系统，结合了海量数据、大模型容量、LLM集成和强化学习（RL）。
- **架构**：音频编码器(0.7B/0.2B) + 音频适配器 + CTC解码器 + LLM解码器(7B/0.6B)
- **训练范式**：5阶段SFT + 上下文SFT + FunRL（GRPO强化学习框架）

**性能指标：**

| 测试场景 | Fun-ASR (7.7B) | Fun-ASR-nano (0.8B) |
| :--- | :--- | :--- |
| AISHELL-1 WER | 1.22% | 1.80% |
| 工业数据集平均WER | 12.70% | 16.72% |
| 复杂背景噪声WER | 11.45% | 14.59% |

**技术特点：**
- RL有效抑制LLM幻觉
- 热词定制：基于RAG机制，召回率可达0.97+
- 流式能力：支持低延迟实时处理


### 3.8 Mega-ASR（面向复杂声学环境的鲁棒ASR）

**方法思路：**
Mega-ASR旨在解决真实世界复合声学环境下的鲁棒性问题。
- **VOICES-IN-THE-WILD-2M数据集**：模拟54种复合场景，240万条合成音频
- **A2S-SFT**：三阶段渐进式微调
- **DG-WGPO**：双粒度WER门控策略优化，抑制幻觉和漏句
- **环境感知路由**：轻量级LoRA路由器，自动判断是否使用鲁棒权重

**性能指标：**

| 基准测试 | Qwen3-ASR-1.7B | Mega-ASR | 相对提升 |
| :--- | :--- | :--- | :--- |
| 鲁棒基准平均WER | 7.93% | **6.70%** | 降低 15.5% |
| NOIZEUS 0dB 极端噪声 | 23.97% | **19.80%** | 降低 17.4% |
| 复合场景平均WER | 5.39% | **2.73%** | 降低 49.3% |

**优势**：复合声学环境SOTA，有效减少LLM幻觉，环境路由保证干净语音性能不退化

## 五、辅助方案/框架对比

以下为辅助性方案与框架，不作为独立ASR模型进行精度对比，但在特定场景下具有重要价值：

| 方案 | 类型 | 关键指标 | 核心特色与定位 | 论文/代码 |
| :--- | :--- | :--- | :--- | :--- |
| **sherpa-onnx** | 部署推理框架 | 跨平台ONNX推理 | 统一部署框架，支持Qwen3-ASR/SenseVoice/Whisper等，12种语言API | [GitHub](https://github.com/k2-fsa/sherpa-onnx) |
| **FM-Refiner** | 潜层增强模块 | 9.2% WER相对降低 | 即插即用流匹配模块，无需重训ASR，3步ODE积分 | 论文中方法 |
| **SpecASR** | 推理加速框架 | **3.79×加速** | 投机解码专用优化，无损加速LLM-ASR，音频条件化特性利用 | 论文阶段，暂未开源 |
| **SynTTS-Commands** | 数据生成方案 | 中英文99.5%/98%准确率 | CosyVoice 2 TTS合成，384K条语料，解决TinyML数据瓶颈 | [GitHub](https://github.com/lugan113/SynTTS-Commands-Official) |

### 辅助方案详细说明：

**sherpa-onnx**
- 基于ONNX Runtime的轻量化语音处理推理框架
- 支持将Qwen3-ASR、SenseVoice、Whisper等多种ASR模型统一为ONNX格式
- 跨平台支持x86、ARM、RISC-V，覆盖Linux/Windows/macOS/Android/iOS
- 提供C++、Python、Rust、Java等12种编程语言API
- 集成CUDA、OpenVINO、RKNN等硬件加速引擎

**FM-Refiner**
- 基于流匹配（Flow Matching）的潜层增强模块
- 直接作用于预训练CTC-ASR编码器的输出潜变量，无需重新训练或微调ASR模型
- 推理仅需3步ODE积分，轻量高效
- 实验验证：无SE前端时相对WER降低9.2%，配合Conv-TasNet降低6.8%
- 弥合波形增强与ASR识别目标不一致的瓶颈

**SpecASR**
- 专门为LLM-based ASR设计的投机解码加速框架
- 核心观察：ASR解码是音频条件化的，小模型与大模型输出高度对齐
- 三大创新：自适应单序列预测 + 草稿序列回收 + 两遍稀疏token树
- 在TinyLlama→Vicuna-13B配置下实现**3.79倍加速**，且**无损精度**
- 消融实验验证各模块均有效降低延迟

**SynTTS-Commands**
- 基于CosyVoice 2 TTS模型合成的大规模语音命令数据集
- 数据规模：384,621条语音，111.33小时，35.76GB
- 覆盖855+5994说话人嵌入，中英文48个命令类别
- 基准测试：DS-CNN达英文99.46%、中文97.18%准确率
- 证明TTS合成数据可有效替代人工录音，解决TinyML数据瓶颈

## 九、最终方案推荐
- 先跑两个较为轻量的模型：SenseVoice-Small和Paraformer-zh，得到基本的ASR结果，实现对论文方案的大致复现
- 冲击高分：在两个轻量模型都跑通，且想要进一步追求正确率，可以使用Qwen3-ASR-0.6B来作为备选方案
- 可使用FireRedASR-AED-L进行高精度对照，但该模型较大，可以只是进行对比，不必部署
- 在保证正确率，精度和鲁棒性的前提下，使用sherpa-onnx等辅助模型工具进一步轻量化模型，加快推理速度
